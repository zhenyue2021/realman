02 · 模块间交互关系
🔑 认证权限流
1
前端POST /auth/login，携带用户名密码
2
Spring Security 验证，生成 Access Token + Refresh Token，写入 Redis
3
Gateway 拦截所有请求，JWT Filter 验签 + 权限检查
4
多租户隔离：TenantContext 注入，SQL自动追加 tenant_id
5
登出时 Redis 黑名单 标记 Token 失效
📡 设备 ↔ 云端通信流
1
设备 MQTT Connect 到 EMQX，携带 clientId=deviceId
2
EMQX Webhook 通知后端设备上下线，更新 Redis 在线状态
3
后端 Publish device/{id}/config 下发配置；设备 Subscribe 后应用
4
设备 Publish device/{id}/status 上报遥测数据
5
后端接收后存库，通过 WebSocket STOMP 推送前端实时展示
🚀 OTA 固件升级流
1
管理员上传固件包，后端校验MD5后存入 MinIO
2
选择目标设备（单个/批量），创建 ota_task 记录
3
Publish device/{id}/ota，携带下载URL + MD5 + 版本号
4
设备下载 → MD5校验 → 安装，上报 device/{id}/ota/report
5
后端更新任务状态，WebSocket 推送升级进度到前端
💳 订阅 & 支付流
1
用户选择订阅方案，后端创建 订阅订单，状态=待支付
2
根据用户地区选择通道：国内→微信/支付宝，海外→Stripe
3
支付回调 Notify 接口，验签后更新订单状态为已支付
4
异步任务同步订单信息至 钉钉 指定群/用户
5
XXL-JOB 定时扫描到期订阅，发送续费提醒
🔧 工单生命周期流
1
管理平台创建工单：配置是否需验收、超时时间、指派人员
2
WebSocket 推送工单通知到遥操平台运维人员
3
运维人员在遥操平台开始工单，状态机流转：待处理→处理中
4
处理完成提交工单，若配置验收则流转→待验收
5
超时未处理触发 XXL-JOB 策略：升级通知/自动关闭
🎥 远程支持 / 音视频流
1
运维人员发起通话，后端生成 WebRTC 信令房间
2
通过 WebSocket 交换 SDP Offer/Answer + ICE候选
3
P2P 通话建立，同步录制音频流到 MinIO
4
通话结束，调用 STT API 转录录音为文字
5
调用 LLM API 提取AI摘要，写入通话记录表
03 · MQTT Topic 规划 & 数据库核心表
📡 MQTT Topic 规范
device/{id}/config 配置参数下发 ↓云→设备
device/{id}/ota OTA升级指令 ↓云→设备
device/{id}/command 实时控制指令 ↓云→设备
device/{id}/status 状态+遥测上报 ↑设备→云
device/{id}/log 操作日志上报 ↑设备→云
device/{id}/ota/report OTA升级结果 ↑设备→云
🗄️ 核心数据库表
sys_user / sys_role / sys_menu 用户权限 RBAC
iot_device / iot_device_config 设备 & 配置
iot_device_telemetry 遥测时序数据
iot_firmware / iot_ota_task 固件 & 升级任务
sub_plan / sub_order 订阅方案 & 订单
work_order / work_order_config 工单 & 合规配置
call_record / payment_order 通话记录 & 支付单
⚡ Redis Key 规范
token:access:{userId} Access Token 缓存
token:blacklist:{token} 登出黑名单
device:online:{deviceId} 设备在线状态
device:status:{deviceId} 最新遥测快照
rate:limit:{userId}:{api} 接口限流计数
order:pay:expire:{orderId} 支付超时 TTL
⏰ XXL-JOB 定时任务
设备离线检测 每2分钟扫描心跳
订阅到期提醒 每天09:00推钉钉
工单超时检查 每5分钟检查
遥测数据归档 每天00:00冷归档
OTA任务超时 每15分钟检查失败
支付订单过期 Redis TTL + 定时确认
