## 1. 模块概述

### 1.1 功能边界

| 子模块                 | 操作角色              | 核心职责                                                                                       |
| ------------------------ | ----------------------- | ------------------------------------------------------------------------------------------------ |
| **工单合规配置** | 平台管理员            | 定义工单执行规则：任务类型、时限策略、验收要求、超时处理方式；工单创建时必须绑定一条已启用配置 |
| **工单管理**     | 平台管理员 / 遥操人员 | 管理员创建工单并绑定主控设备与机器人；遥操人员通过主控端领取并执行工单；管理员负责审核与关闭   |

### 1.2 角色定义

| 角色                         | 说明                                                                     |
| ------------------------------ | -------------------------------------------------------------------------- |
| **平台管理员**         | 维护合规配置、创建工单、审核工单结果、关闭工单                           |
| **遥操人员（操作员）** | 登录主控设备，在主控端查看并领取工单，执行任务后提交，异常时填写超时原因 |

### 1.3 与 IoT 模块的关系

* 工单绑定的**主控设备**对应 `iot_device.device_type = 2`
* 工单绑定的**机器人**对应 `iot_device.device_type = 1`
* 操作员登录记录复用 `iot_controller_login_log` 表
* 授权关系复用 `iot_device_auth` 表（确保主控与机器人已完成授权绑定）

## 2. 工单合规配置

### 2.1 基本信息字段

| 字段       | 类型     | 必填     | 说明                                 |
| ------------ | ---------- | ---------- | -------------------------------------- |
| 配置ID     | Long   | 系统生成 | 唯一标识，雪花ID                       |
| 配置名称   | String   | 是       | 合规配置显示名称，如“标准巡检规则” |
| 任务类型   | Enum     | 是       | 巡检 / 调试 / 维护 / 其他（可扩展）  |
| 任务说明   | String   | 否       | 对该合规配置的详细描述               |
| 任务等级   | Enum     | 否       | 普通 / 紧急 / 高优先级               |
| 所属代理商 | String   | 是       | 代理商名称/ID                        |
| 所属企业   | String   | 否       | 企业名称/ID                          |
| 创建时间   | DateTime | 系统生成 | 格式 Y/M/D H:M:S                     |
| 修改时间   | DateTime | 系统生成 | 格式 Y/M/D H:M:S                     |
| 状态       | Enum     | 系统维护 | 未启用 / 已启用                      |

### 2.2 任务规则配置项

#### 2.2.1 工单超时提醒

| 配置项         | 说明                                                                            |
| ---------------- | --------------------------------------------------------------------------------- |
| 开关           | 启用 / 禁用                                                                     |
| **禁用** | 不发送任何提醒                                                                  |
| **启用** | 在距工单结束时间 ​**X**​（H:M:S）时，向当前执行工单的主控端推送提醒通知 |
| 配置参数       | 提前提醒时长 X，格式 H:M:S，如 `00:30:00`（提前30分钟）                     |

> 提醒方式：主控端弹窗通知 + WebSocket 推送前端管理界面

#### 2.2.2 工单提交时限

| 配置项         | 说明                                                                    |
| ---------------- | ------------------------------------------------------------------------- |
| 开关           | 启用 / 禁用                                                             |
| **禁用** | 无提交时间限制，工单失效后仍可提交（进入超时异常流程）                  |
| **启用** | 提交时间必须 ≤ 工单结束时间（`plan_end_time`），超时后提交接口拦截 |

#### 2.2.3 工单验收结果

| 配置项         | 说明                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------- |
| 开关           | 启用 / 禁用                                                                               |
| **禁用** | 无需人工验收，遥操人员提交即为终态（`SUBMITTED` 直接视为完成）                        |
| **启用** | 遥操人员提交后，状态为 `SUBMITTED`，等待管理员审核；审核结果为**合格 / 不合格** |

#### 2.2.4 超时提交处理（工单异常）

适用于：遥操人员在 `plan_end_time` 之后仍尝试提交的情况。

| 配置项           | 说明                                                                 |
| ------------------ | ---------------------------------------------------------------------- |
| 触发条件         | 实际提交时间 > `plan_end_time`                                   |
| 处理方式         | 工单状态置为 `TIMEOUT`，要求操作员填写超时原因                   |
| 超时原因         | 选择枚举（用户原因/节假日/设备故障）或自定义填写（由 `overtime_reason_enum/overtime_reason_desc` 约束与说明） |

#### 2.2.5 超时未提交（自动关闭）

| 配置项         | 说明                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------ |
| 开关           | 启用 / 禁用                                                                                                |
| 触发条件       | 当前时间 ≥ `plan_end_time`，且工单仍未提交                                                            |
| **禁用** | 仅由定时任务将状态置为 `TIMEOUT`，需人工介入处理                                                       |
| **启用** | 定时任务在超时检测到后，自动将工单置为 `CLOSED`，`timeout_reason` 写入默认值：“操作员超时未完成” |
| 配置参数       | 判定宽限期 X（H:M:S）：当前时间超过 `plan_end_time + X`（即 `auto_close_offset`）后才触发自动关闭（避免误判）                   |

### 2.3 设备与人员范围约束（可选）

| 配置项                 | 说明                                                  |
| ------------------------ | ------------------------------------------------------- |
| 允许绑定的主控设备范围 | 限制工单只能绑定授权给当前用户所属企业的主控设备 |
| 允许绑定的机器人范围   | 限制工单只能绑定给当前用户所属企业的机器人设备   |
| 允许执行的操作员范围   | 限制只有工单任务已生效且未开启的工单              |

### 2.4 状态管理规则

| 状态             | 允许操作               |
| ------------------ | ------------------------ |
| **未应用** | 查看、编辑、删除       |
| **已应用** | 仅查看，不可编辑和删除 |

> 已启用的配置被工单引用后不得修改，保证历史工单数据一致性。若需修改，建议新建配置并禁用旧配置。

### 2.5 功能操作

| 操作      | 说明                                                                   |
| ----------- | ------------------------------------------------------------------------ |
| 新增      | 创建合规配置，初始状态为未启用                                         |
| 编辑      | 仅未启用状态可编辑                                                     |
| 删除      | 仅未启用状态可删除；已被工单引用的配置不可删除（提示“配置已被引用”） |
| 启用/禁用 | 管理员手动切换状态；禁用时检查是否有进行中工单                         |
| 导出      | 按当前筛选条件导出 Excel                                               |

### 2.6 查询与筛选条件

| 条件                | 说明                   |
| --------------------- | ------------------------ |
| 租户（代理商/企业） | 按所属代理商或企业筛选 |
| 应用状态            | 已应用 / 未应用        |

## 3. 工单管理

### 3.1 工单基本信息（管理员创建时填写）

| 字段         | 类型     | 必填     | 说明                                                                            |
| -------------- | ---------- | ---------- | --------------------------------------------------------------------------------- |
| 工单ID       | String   | 系统生成 | 唯一标识，UUID                                                                  |
| 工单任务名称 | String   | 是       | 工单标题                                                                        |
| 代理商       | String   | 是       | 代理商代码 / 名称                                                               |
| 所属部门     | String   | 否       | 上级部门                                                                        |
| 合规配置     | String   | 是       | 绑定一条合规配置（规则）ID；规则有工单绑定时标记为**已应用** |
| 工单开始时间 | DateTime | 是       | `plan_start_time`，格式 Y/M/D H:M:S                                         |
| 工单结束时间 | DateTime | 是       | `plan_end_time`，即工单失效时间，格式 Y/M/D H:M:S                           |
| 币种         | String   | 否       | 支持根据IP定位自动切换（如CNY/USD）                                             |
| 计费         | -        | 否       | 包含单价、总价（可选）                                                          |
| 备注         | String   | 否       | 补充说明                                                                        |
| 工单状态     | Enum     | 系统维护 | 见第 4 章                                                                       |
| 创建人       | String   | 系统生成 | 当前登录的管理员账号                                                            |
| 创建时间     | DateTime | 系统生成 | 格式 Y/M/D H:M:S                                                                |

### 3.2 绑定设备（管理员创建工单时指定）

#### 主控设备（必填，单台）

对应 `iot_device.device_type = 2`

| 字段     | 类型   | 说明                         |
| ---------- | -------- | ------------------------------ |
| 设备ID   | String | `iot_device.id`          |
| 设备名称 | String | `iot_device.device_name` |
| 设备编号 | String | `iot_device.device_code` |

#### 机器人（必填，可多台）

对应 `iot_device.device_type = 1`

| 字段       | 类型   | 说明                         |
| ------------ | -------- | ------------------------------ |
| 机器人ID   | String | `iot_device.id`          |
| 机器人名称 | String | `iot_device.device_name` |
| 机器人编号 | String | `iot_device.device_code` |

> 合规配置若配置了设备范围约束，创建工单时只可选择约束范围内的设备。

### 3.3 开启信息（遥操人员点击“开始”时自动记录）

| 字段           | 类型     | 说明                                                     |
| ---------------- | ---------- | ---------------------------------------------------------- |
| 操作员ID       | String   | 当前登录主控端的用户 ID                                  |
| 操作员姓名     | String   | 当前用户姓名                                             |
| 操作员联系方式 | String   | 当前用户手机号                                           |
| 实际开启时间   | DateTime | 遥操人员点击“开始”时的服务端时间                       |
| 登录记录ID     | String   | 关联 `iot_controller_login_log.id`，用于追溯登录会话 |

### 3.4 工单补充信息（开启工单后填写，设备信息快照）

遥操人员点击开始后，须确认本次实际使用的设备信息（与计划绑定的设备可能存在差异），按创建工单时绑定的数量分组展示，每组至少填写一条。

#### 主控端实际设备快照（按绑定数量分组）

| 字段     | 类型   | 必填 | 说明                                                    |
| ---------- | -------- | ------ | --------------------------------------------------------- |
| 设备ID   | String | 是   | 实际使用主控设备 ID                                     |
| 设备名称 | String | 是   | 实际使用主控设备名称                                    |
| 设备编号 | String | 是   | 实际使用主控设备编号                                    |
| 固件版本 | String | 否   | 开始时从 `iot_device.firmware_version` 自动读取快照 |

#### 机器人实际设备快照（按绑定数量分组）

| 字段       | 类型   | 必填 | 说明                                                    |
| ------------ | -------- | ------ | --------------------------------------------------------- |
| 机器人ID   | String | 是   | 实际使用机器人 ID                                       |
| 机器人名称 | String | 是   | 实际使用机器人名称                                      |
| 机器人编号 | String | 是   | 实际使用机器人编号                                      |
| 固件版本   | String | 否   | 开始时从 `iot_device.firmware_version` 自动读取快照 |

#### 参与人员信息

| 字段     | 类型   | 必填 | 说明                   |
| ---------- | -------- | ------ | ------------------------ |
| 姓名     | String | 是   | 操作员姓名             |
| 联系方式 | String | 是   | 操作员手机号           |
| 角色     | String | 是   | 遥操员 / 监督员 / 其他 |

### 3.5 提交信息

| 字段     | 类型     | 说明                                                     |
| ---------- | ---------- | ---------------------------------------------------------- |
| 提交时间 | DateTime | 遥操人员点击“提交”时的服务端时间                       |
| 佐证图片 | List     | 合规配置要求上传时为必填，存 `work_order_attachment` |

### 3.6 超时原因（超时异常时填写）

| 字段     | 类型   | 必填     | 说明                                                                     |
| ---------- | -------- | ---------- | -------------------------------------------------------------------------- |
| 超时原因 | String | 是       | 操作员手动填写，字数限制取自合规配置（默认100字）                        |
| 原因来源 | Enum   | 系统维护 | `USER`=操作员填写；`SYSTEM`=管理员关闭时系统默认写入“用户原因” |

### 3.7 审核信息（管理员审核时填写）

| 字段     | 类型     | 说明                               |
| ---------- | ---------- | ------------------------------------ |
| 审核人   | String   | 管理员账号                         |
| 审核时间 | DateTime | 格式 Y/M/D H:M:S                   |
| 审核结果 | Enum     | `PASS`=合格；`FAIL`=不合格 |
| 审核备注 | String   | 可选，审核说明                     |

### 3.8 工单查询与筛选条件

| 条件              | 说明                                                  |
| ------------------- | ------------------------------------------------------- |
| 工单ID / 任务名称 | 精确或模糊匹配                                        |
| 代理商 / 企业     | 按租户筛选                                            |
| 工单状态          | 未开始 / 已开始 / 已提交 / 已完成 / 超时异常 / 已关闭 |
| 主控设备          | 按 controller_id / device_code                        |
| 机器人            | 按 robot_id / device_code                             |
| 操作员            | 按 operator_id / operator_name                        |
| 计划时间范围      | plan_start_time / plan_end_time 区间                  |
| 实际开始时间范围  | actual_start_time 区间                                |
| 提交时间范围      | submit_time 区间                                      |
| 审核时间范围      | audit_time 区间                                       |
| 是否超时          | timeout_flag                                          |

## 4. 工单状态流转

### 4.1 状态定义

| 状态值            | 中文名   | 触发条件                                                  |
| ------------------- | ---------- | ----------------------------------------------------------- |
| `NOT_STARTED` | 未开始   | 工单创建完成，无人开启                                    |
| `IN_PROGRESS` | 已开始   | 遥操人员点击“开始”，记录开启信息                        |
| `SUBMITTED`   | 已提交   | 遥操人员在 `plan_end_time` 前点击“提交”             |
| `COMPLETED`   | 已完成   | 管理员审核（合格/不合格均进入此状态）                     |
| `TIMEOUT`     | 超时异常 | 到达 `plan_end_time` 仍未提交；等待操作员填写超时原因 |
| `CLOSED`      | 已关闭   | 管理员手动关闭，或系统自动关闭（合规配置启用自动关闭时）  |

### 4.3 状态流转规则

| 当前状态          | 目标状态          | 触发角色 | 前置条件                                                  | 补充说明                                 |
| ------------------- | ------------------- | ---------- | ----------------------------------------------------------- | ------------------------------------------ |
| `NOT_STARTED` | `IN_PROGRESS` | 遥操人员 | 当前时间 ≥ `plan_start_time`                         | 填写补充信息                             |
| `NOT_STARTED` | `CLOSED`      | 管理员   | 无                                                        | 需填关闭原因                             |
| `IN_PROGRESS` | `SUBMITTED`   | 遥操人员 | 当前时间 ≤ `plan_end_time`（合规配置启用提交时限时） | 可上传佐证图片                           |
| `IN_PROGRESS` | `TIMEOUT`     | 系统自动 | 当前时间 > `plan_end_time`                            | 等待操作员填写超时原因                   |
| `IN_PROGRESS` | `CLOSED`      | 管理员   | 无                                                        | 需填关闭原因                             |
| `SUBMITTED`   | `COMPLETED`   | 管理员   | 合规配置启用验收                                          | 审核结果：PASS / FAIL                    |
| `SUBMITTED`   | `COMPLETED`   | 系统自动 | 合规配置**禁用**验收                                | 提交即完成，result=PASS                  |
| `TIMEOUT`     | `CLOSED`      | 管理员   | 无                                                        | 若未填超时原因，系统自动写入“用户原因” |
| `TIMEOUT`     | `CLOSED`      | 系统自动 | 合规配置启用自动关闭                                      | 写入默认超时原因                         |

## 5. 主控端工单交互流程

### 5.1 工单列表展示规则

遥操人员登录主控设备后，系统自动筛选并展示：

| 筛选条件     | 说明                                                                                                         |
| -------------- | -------------------------------------------------------------------------------------------------------------- |
| 主控设备匹配 | `work_order_device.device_id = 当前主控设备ID`，`device_type = CONTROLLER`                           |
| 工单状态     | `NOT_STARTED`                                                                                            |
| 时间窗口     | `plan_start_time ≤ 当前时间 + 24小时`（提前预览），**或** 已过 `plan_start_time` 但仍无人开启 |
| 排序         | 按 `plan_start_time` 升序                                                                                |

### 5.2 完整操作时序

text

```
遥操人员登录主控设备
      │
      ▼
    查看工单列表
    （状态=未开始，按时间升序排列）
      │
      ▼
    选择工单 → 点击【开始】
      │
      ├─ 服务端校验：当前时间 ≥ plan_start_time
      ├─ 工单状态 → IN_PROGRESS
      ├─ 记录：operator_id / operator_name / operator_phone
      ├─ 记录：actual_start_time = now()
      ├─ 记录：login_log_id（关联当前主控登录记录）
      │
      ▼
    填写工单补充信息
    （实际主控端设备快照 / 机器人快照 / 参与人员）
      │
      ▼

      │
```

## 6. 定时任务设计

### 6.1 任务列表

| 任务名称               | 执行周期 | 触发条件                                                                                      | 执行逻辑                                                                                                            |
| ------------------------ | ---------- | ----------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **工单超时提醒** | 每分钟   | `plan_end_time - alert_offset ≤ now < plan_end_time` 且状态为 `IN_PROGRESS`         | 向主控端推送超时提醒（WebSocket + 消息通知），每个工单​**只推送一次**​（用 Redis 标记已推送）               |
| **工单超时检测** | 每分钟   | `now ≥ plan_end_time` 且状态为 `IN_PROGRESS` 或 `NOT_STARTED`                    | 将状态置为 `TIMEOUT`，`timeout_flag = true`                                                                 |
| **超时自动关闭** | 每分钟   | `now ≥ plan_end_time + auto_close_offset` 且状态为 `TIMEOUT` 且合规配置启用自动关闭 | 将状态置为 `CLOSED`，写入默认 `timeout_reason = “操作员超时未完成”`，`timeout_reason_source = SYSTEM` |

### 6.2 避免重复推送

超时提醒使用 Redis 记录已推送状态：

text

```
Key:   work-order:alert-sent:{workOrderId}
Value: 1
TTL:   工单结束时间 + 1小时
```

* 非超管用户按 `iot_device_auth`（`subject_id` = 当前用户/租户）过滤可见工单
* 工单列表查询时关联 `iot_device_auth` 确保只返回有权限的主控设备下的工单
