**版本**: 1.0  
**更新日期**: 2025-03  
**建设单位**: 睿尔曼智能科技（北京）有限公司

## 1. 概述

### 1.1 项目简介

Realman-boot 是基于 **JeecgBoot 3.x** 的定制化企业级应用平台，采用 **Spring Boot 3**、**Java 21** 技术栈，支持单体与微服务两种部署形态。项目面向睿尔曼业务场景，在通用系统管理（用户、组织、权限、租户、字典、消息等）基础上，扩展了 **IoT 设备管理**（机器人/主控设备、MQTT 接入、OTA 升级、工单与合规）等服务。

### 1.2 设计目标

- **分层清晰**：API / Biz / Start 分层，便于维护与扩展。
- **可拆分部署**：系统服务与 IoT 服务可独立进程，也可按需接入 Nacos 做服务发现。
- **安全可控**：统一 Shiro + JWT 鉴权；IoT 设备侧通过 EMQX HTTP Auth/ACL + 设备密钥 + AES 加密通信。
- **技术统一**：MyBatis-Plus、Druid、Redis、MySQL 等统一技术选型，支持多数据源与国产库。

## 4. 分层架构

### 4.1 系统服务（realman-boot）


| 层级    | 职责                    | 主要包/组件                                                              |
| ----- | --------------------- | ------------------------------------------------------------------- |
| 接入层   | 登录、鉴权、参数解析、统一响应       | Controller（LoginController、SysUserController、SysDepartController 等） |
| 服务层   | 业务逻辑、事务、多表协调          | Service 接口与实现（org.jeecg.modules.system.service）                     |
| 数据访问层 | 持久化、多数据源、租户过滤         | Mapper (MyBatis-Plus)、部分 MiniDao                                    |
| 支撑层   | 权限、日志、字典、消息、定时任务、文件上传 | Shiro、AOP、XXL-JOB、CommonController、OSS                              |


**安全链路**：请求 → Shiro Filter（JwtFilter / 匿名白名单）→ Controller → @RequiresPermissions 等 → Service → Mapper。会话可存 Redis（shiro-redis），Token 由 JWT 生成与校验。

### 4.2 IoT 服务（realman-iot）


| 层级     | 职责                                       | 主要包/组件                                                                                                            |
| ------ | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| 接口层    | REST API、参数校验、统一封装                       | Controller（MasterDeviceController、RobotDeviceController、DeviceAuthController、OtaController、WorkOrderController 等） |
| 业务层    | 设备 CRUD、配置同步、监控、重启、紧急停机、登录解析、工单、OTA、操作记录 | Service 接口与实现（device.service）                                                                                     |
| MQTT 层 | 设备连接鉴权、消息路由、上行处理、下行发布                    | MqttAuthController（HTTP 回调）、MqttMessageDispatcher、各类 Handler、MqttPublisher                                        |
| 安全/加密  | 设备密钥、AES 加密、ACL                          | DeviceSecretService、CommandEncryptService                                                                         |
| 数据层    | 设备/配置/状态/日志/工单/OTA/授权                    | Mapper、entity、Redis（在线集合、状态缓存等）                                                                                   |


**设备鉴权**：设备使用 `deviceCode` + `MD5(deviceCode)` 连接 EMQX；EMQX 通过 HTTP 回调 `/internal/mqtt/auth`、`/internal/mqtt/acl` 与平台校验；业务报文按设备 AES-256-CBC 加解密。

## 5. 模块说明

### 5.1 realman-boot-base-core

- **定位**：整个项目的公共核心，被系统模块与 IoT 模块共同依赖。
- **主要内容**：
  - **安全**：Shiro 配置、JWT Token、JwtFilter、ShiroRealm、IgnoreAuth、ResourceCheckFilter。
  - **持久化**：MyBatis-Plus 公共配置、多数据源、租户插件（MybatisPlusSaasConfig）、审计等。
  - **通用**：Result 统一返回、QueryGenerator、异常处理、日志 AOP、BaseCommonService、RedisUtil、JwtUtil、加解密工具等。
  - **Web**：Actuator、Knife4j/OpenAPI、WebSocket 过滤等。

### 5.2 realman-module-system

- **定位**：系统管理能力，支撑所有业务对用户、组织、权限、租户、字典、消息等的基础需求。
- **子模块**：
  - **realman-system-api**：对外接口契约；realman-system-local-api 供单体直接依赖；realman-system-cloud-api 供微服务 Feign 等调用。
  - **realman-system-biz**：用户、部门、角色、权限、菜单、租户、字典、公告、操作日志、数据日志、在线用户、数据源、填值/校验规则、岗位、部门角色/权限、评论、文件上传、定时任务等完整实现。
  - **realman-system-start**：组装 biz + demo，提供 RealmanSystemApplication 单体启动入口。

### 5.3 realman-boot-iot

- **定位**：IoT 设备全生命周期与遥操相关业务，包括设备管理、MQTT 消息、OTA、工单与合规。
- **子模块**：
  - **realman-boot-iot-api**：REST 接口定义（Controller）、DTO/VO、对外 API 契约；不含 MQTT 与复杂业务实现。
  - **realman-boot-iot-biz**：设备/配置/状态/日志/授权/工单/合规/OTA 等业务实现；MQTT 订阅与发布、消息分发（MqttMessageDispatcher）、各类 Topic Handler（上下线、状态、配置 ACK、重启 ACK、OTA 进度、操作日志、主控关联设备、摄像头流等）；设备密钥与 AES 加密；Redis 在线集合与状态缓存；MinIO 固件存储。
  - **realman-boot-iot-start**：启动包，依赖 api + biz，启用 Nacos Discovery等，扫描 org.jeecg 以加载 base-core 安全与工具。

## 6. 数据与集成架构

### 6.1 数据存储


| 存储         | 用途           | 说明                                           |
| ---------- | ------------ | -------------------------------------------- |
| MySQL      | 业务主库         | 用户、组织、权限、租户、字典、公告、日志、设备、配置、工单、OTA、授权等；       |
| Redis      | 缓存、会话、锁、设备在线 | 会话（Shiro）、缓存、分布式锁（Redisson）、设备在线集合与实时状态（IoT） |
| MinIO / 本地 | 文件、OTA 固件    | 上传文件、OTA 固件包；配置项切换                           |


### 6.2 外部系统与中间件


| 组件                       | 用途          | 说明                                                     |
| ------------------------ | ----------- | ------------------------------------------------------ |
| EMQX                     | MQTT Broker | 设备连接、Topic 路由；HTTP Auth/ACL 回调平台；IoT 服务订阅设备 Topic、下发指令 |
| Nacos                    | 服务发现/配置     | 微服务注册与发现；IoT 可单独注册                                     |
| XXL-Job                  | 分布式定时任务     | 替代或补充内置 Quartz                                         |
| RabbitMQ / RocketMQ（待实现） | 消息队列        | 业务解耦、异步处理                                              |


### 6.3 安全与加密

- **Web 端**：Shiro + JWT；密码 AES 传输、加盐存储；。
- **IoT 设备**：连接层 EMQX HTTP Auth/ACL；设备密码 MD5(deviceCode)；业务 payload 使用设备密钥派生的 AES-256-CBC 加密。
- **多租户**：租户隔离（MybatisPlusSaasConfig 等）；

## 8. 关键设计说明

### 8.1 统一响应与异常

- **系统/IoT 接口**：成功返回 `Result<T>`（success、message、code、result、timestamp）或项目内 `ApiResult<T>`（code、message、data）；失败由全局异常处理（如 JeecgBootExceptionHandler）统一封装。
- **分页**：使用 MyBatis-Plus `IPage`/`Page`，统一包含 records、total、current、size 等。

### 8.2 多数据源与多租户

- **多数据源**：dynamic-datasource，可在业务中切换数据源。
- **多租户**：通过 MybatisPlusSaasConfig 等对系统表进行租户过滤；IoT 模块当前排除该配置，按单租户或业务自行扩展。

### 8.3 IoT 与系统协同

- **鉴权**：IoT 的 REST 接口复用 base-core 的 Shiro + JWT，与系统共用 Token 体系；设备侧仅 MQTT + EMQX 回调，不走 JWT。
- **数据**：设备、授权、工单等存 IoT 库表；用户、部门、权限等由系统服务提供；若前后端一体部署，前端可同时调 8080 与 8085。
